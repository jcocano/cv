<ProjectSection labelEs="decisiones" labelEn="decisions">
  <Lang lang="es">

**Aislamiento por cluster, no por namespace.** Namespaces hubiesen sido más baratos pero comparten el control plane. Optamos por clusters EKS separados por entorno para que un fallo no pueda saltar la barrera de seguridad del Kubernetes API.

**Terragrunt sobre Terraform plano.** DRY de configuración, módulos reutilizables y locking de estado por entorno. Cada cluster es un import declarativo, no un copiar-pegar de HCL.

**Networking privado avanzado.** VPC peering controlado, egress restringido y secrets gestionados de forma centralizada — la superficie de exposición se redujo a lo estrictamente necesario.

  </Lang>
  <Lang lang="en">

**Isolation per cluster, not per namespace.** Namespaces would've been cheaper but share the control plane. We chose separate EKS clusters per environment so a failure can't jump the Kubernetes API security boundary.

**Terragrunt over flat Terraform.** DRY config, reusable modules, and per-environment state locking. Each cluster is a declarative import, not an HCL copy-paste.

**Advanced private networking.** Controlled VPC peering, restricted egress, and centralized secret management — exposure surface reduced to the strict minimum.

  </Lang>
