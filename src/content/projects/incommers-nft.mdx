<ProjectSection labelEs="decisiones" labelEn="decisions">
  <Lang lang="es">

**Merkle Tree para whitelist.** Solo guardamos el root hash on-chain. El cliente envía la prueba con cada mint y el contrato la verifica. Resultado: costo de almacenamiento on-chain constante independientemente del tamaño de la whitelist.

**Optimización de slots.** Los structs se empaquetaron para minimizar SSTOREs, y movimos lógica frecuente a funciones puras donde fue posible.

**Web responsiva integrada.** Ethers.js + Next.js para una capa de UI que conecta wallets, genera la prueba localmente y dispara mints sin que el usuario tenga que entender la criptografía detrás.

  </Lang>
  <Lang lang="en">

**Merkle Tree whitelist.** Only the root hash lives on-chain. The client submits the proof with each mint and the contract verifies it. Result: constant on-chain storage cost regardless of whitelist size.

**Storage slot packing.** Structs were packed to minimize SSTOREs, and frequently called logic moved to pure functions where possible.

**Tightly integrated responsive web.** Ethers.js + Next.js for a UI layer that connects wallets, generates the proof locally, and triggers mints without the user needing to understand the crypto behind it.

  </Lang>
